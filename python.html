17. Programmation Orientée Objet (POO) en Python

La Programmation Orientée Objet permet d’organiser le code en objets avec des attributs (données) et des méthodes (fonctions). En Python, on utilise les classes pour créer des objets.

Exemple de classe simple :

class Personne:
    def __init__(self, nom, age):
        self.nom = nom
        self.age = age

    def saluer(self):
        return f"Bonjour, je m'appelle {self.nom} et j'ai {self.age} ans."

# Création d'un objet
eric = Personne("Eric", 20)
print(eric.saluer())

Résultat : Affiche : Bonjour, je m'appelle Eric et j'ai 20 ans.

Héritage (subclasses)

class Employe(Personne):
    def __init__(self, nom, age, poste):
        super().__init__(nom, age)
        self.poste = poste

    def travailler(self):
        return f"{self.nom} travaille en tant que {self.poste}."

anna = Employe("Anna", 25, "Développeuse")
print(anna.saluer())
print(anna.travailler())

Résultat :

  • Bonjour, je m'appelle Anna et j'ai 25 ans.
  • Anna travaille en tant que Développeuse.

Méthodes et attributs privés

class Compte:
    def __init__(self):
        self.__solde = 0  # attribut privé

    def depot(self, montant):
        self.__solde += montant

    def get_solde(self):
        return self.__solde

mon_compte = Compte()
mon_compte.depot(100)
print(mon_compte.get_solde())

Résultat : 100 (le solde est protégé et accessible uniquement via la méthode)

Méthodes de classe et statiques

class Calcul:
    @staticmethod
    def addition(a, b):
        return a + b

print(Calcul.addition(5, 3))  # 8

Résultat : 8 (pas besoin de créer un objet)

Conclusion sur la POO en Python

La POO en Python permet de :

  • Structurer le code avec des objets
  • Réutiliser le code via l’héritage
  • Encapsuler les données pour plus de sécurité
  • Créer des programmes plus modulaires et maintenables

Les classes, objets, héritage et encapsulation sont essentiels pour développer des projets Python complexes et professionnels.
